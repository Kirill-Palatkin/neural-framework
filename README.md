# MiniFlow

`MiniFlow` - учебный нейросетевой фреймворк на Python для обучения полносвязных нейронных сетей. Проект сделан специально под лабораторную работу по курсу "Искусственный интеллект": он показывает, как нейросеть можно собирать перечислением слоёв, обучать в несколько строк и применять не только к классическим, но и к необычным задачам.

## Что реализовано

- Построение многослойной нейросети перечислением слоёв через `Sequential([...])`.
- API для работы с данными: `Dataset.from_arrays(...)`, `map(...)`, `shuffle(...)`, `batch(...)`, `split(...)`, `take(...)`.
- Полносвязный слой `Dense`.
- Функции активации: `ReLU`, `LeakyReLU`, `Sigmoid`, `Tanh`, `Softmax`, `Identity`.
- Функции потерь для классификации и регрессии: `MSELoss`, `BinaryCrossEntropy`, `CategoricalCrossEntropy`.
- Алгоритмы и стратегии оптимизации: `SGD`, `MomentumSGD`, `Adam`, а также ограничение нормы градиента через `GradientClipping`.
- Метрики: `accuracy`, `binary_accuracy`, `mse`, `mae`.
- Три примера использования: классификация Iris, регрессия синусоиды и детектор котиков по изображениям.

## Быстрый старт

Нейросеть можно собрать и обучить буквально в несколько строк:

```python
import numpy as np

from miniflow import Adam, CategoricalCrossEntropy, Dataset, Dense, ReLU, Sequential, Softmax

X = np.random.randn(120, 4).astype(np.float32)
y = np.random.randint(0, 3, size=120)

dataset = Dataset.from_arrays(X, y).shuffle(seed=42)
train_dataset, valid_dataset = dataset.split(validation_ratio=0.2)

model = Sequential(
    [
        Dense(16, input_dim=4, initializer="he"),
        ReLU(),
        Dense(16, initializer="he"),
        ReLU(),
        Dense(3),
        Softmax(),
    ]
)

model.fit(
    train_dataset,
    epochs=50,
    batch_size=16,
    optimizer=Adam(learning_rate=0.01),
    loss=CategoricalCrossEntropy(),
    metrics=["accuracy"],
    validation_data=valid_dataset,
)
```

## API для датасетов

Во фреймворке есть компактный набор инструментов для подготовки данных:

```python
dataset = Dataset.from_arrays(X, y)
augmented = dataset.map(lambda x, y: (x / 255.0, y))
shuffled = augmented.shuffle(seed=1)
train_dataset, valid_dataset = shuffled.split(validation_ratio=0.2)

for batch_x, batch_y in train_dataset.batch(batch_size=32):
    ...
```

### 2. Регрессия синусоиды


Что показывает пример:

### 3. Детектор котиков по изображениям

Запуск:

```bash
python examples/cat_detector.py --data-dir data/cats_binary --image-size 32 --epochs 40
```

Что показывает пример:

- Необычный пример для лабораторной работы.
- Загрузку изображений из папок.
- Бинарную классификацию картинок при помощи полносвязной сети.

После обучения скрипт сохраняет:

- веса модели, по умолчанию в `artifacts/cat_detector_weights.npz`
- метаданные модели, по умолчанию в `artifacts/cat_detector_meta.json`

Чтобы заново обучить модель, достаточно снова запустить `cat_detector.py`: веса будут пересчитаны с нуля и сохранены заново.

Чтобы проверить новую картинку уже обученной моделью:

```bash
python examples/predict_cat.py --image path/to/new_image.jpg
```

## Требования к датасету с котиками

Чтобы подготовить свой датасет, создайте папку ровно с двумя подкаталогами классов, например:

```text
data/cats_binary/
├── cat/
│   ├── cat_001.jpg
│   ├── cat_002.jpg
│   └── ...
└── other/
    ├── other_001.jpg
    ├── other_002.jpg
    └── ...
```

Рекомендуемые требования:

- Минимум 150-200 изображений на класс, лучше 300+ на класс.
- Классы должны быть примерно сбалансированы по количеству.
- Поддерживаемые форматы: `.jpg`, `.jpeg`, `.png`, `.bmp`.
- Изображения должны быть разнообразными: разное освещение, фон, масштаб, позы, съёмка дома и на улице.
- Желательно убрать дубликаты, водяные знаки, мемы, скриншоты с текстом и слишком пережатые изображения.
- В класс `other` полезно добавить сложные отрицательные примеры: собак, игрушки, пледы, шерстяные текстуры, тёмные сцены.

Для полносвязной сети большие изображения не нужны. Пример уменьшает их до `32x32`, превращает в вектор и обучает бинарный классификатор. Если хочется повысить качество, можно собирать исходники `64x64` и больше, но перед обучением они всё равно будут приведены к выбранному размеру.

## Почему работа выглядит не как шаблон от LLM

В проект добавлен намеренно нестандартный демонстрационный кейс: детектор котиков, обучаемый на самостоятельно собранном датасете изображений. За счёт этого репозиторий выглядит как мини-фреймворк с авторской демонстрацией, а не как типовая лабораторная только на `Iris`.

## Архитектурные идеи

- Вся математика и backpropagation реализованы вручную на `NumPy`.
- Прямой и обратный проход реализованы отдельно в каждом слое и каждой функции активации.
- Метод `Sequential.fit(...)` берет на себя minibatch-обучение, валидацию, метрики и историю обучения.
- Ограничение градиента сделано как переиспользуемый компонент оптимизации.

## Что можно показать в 5-минутном видео

- Почему образовательный фреймворк написан именно на `NumPy`, а не на полноценной DL-библиотеке.
- Как устроены прямой и обратный проходы в `Dense` и функциях активации.
- Как `Dataset.map`, `shuffle`, `batch` и `split` упрощают обучение.
- Почему пример с котиками делает проект более оригинальным.
- Чем отличаются стратегии оптимизации: `SGD`, `MomentumSGD`, `Adam` и ограничение градиента.

## Ограничения

- Фреймворк рассчитан только на полносвязные сети.
- Детектор котиков сделан нарочно простым и не конкурирует со сверточными архитектурами.
- Для примера `Iris` нужен `scikit-learn`, потому что скрипт использует его встроенный загрузчик датасета.
